7. npm i express body-parser bcryptjs cors dotenv multer-gridfs-storage jsonwebtoken mongoose
       - This is used to install every required essential Node.js packages at once for backend
         
         (1) express 
             - Most popular Node.js web framework
             - whole backend's server will run on this only
             - Let's you easily create APIs and route requests

             Syntax : 
                   const express = require('express');
                   const app = express();


         (2) body-parser 
               - To parse(resolve/divide any sentence or something into parts) the body of request
               - Useful when we send JSON or form data from backend 
               Syntax : 
                    app.use(bodyParser.json());

         (3) bcryptjs 
               - To hash passwords (for security)
               -Plain password is converted into hash so that it's secure in the database
               Syntax :
                     const bcrypt = require('bcryptjs');
                     const hashedPassword = bcrypt.hashSync(password, 10);

          (4) cors (Cross-Origin Resource Sharing)
                -  If frontend and backend is on different domain/ports (like react on 3000, and Node on 5000)
                - CORS allows frontend to connect safely to backend to communicate
                Syntax :
                     const cors = require('cors');
                     app.use(cors());

           (5) dotenv    
                  - To store environment variables (like DB_URI, SECRET_KEY)
                  - Handles sensitive data in a secure way

                  .env file : 
                         PORT=5000
                         MONGO_URI=mongodb+srv://...             

                         require('dotenv').config();

            (6) multer-gridfs-storage
                    - To store file uploads (e.g., images, documents) in mongodb
                    - Specially for GridFS (file storage system of MongoDB)  

                    Syntax : const multer = require('multer-gridfs-storage');

             (7) jsonwebtoken 
                   - used to generate JWL tokens (for login/ authentication)
                   - Tokens are used to access secure route

                   Syntax : 
                   const jwt = require('jsonwebtoken');
                   const token = jwt.sign({ id: user._id }, SECRET_KEY);

             (8) mongoose 
                   - Used to interact with MongoDB easily
                   - This is a ODM (Object Data Modeling) library

                   Syntax :
                         const mongoose = require('mongoose');
                         mongoose.connect(process.env.MONGO_URI);

8.  npm i react-redux @redusjs/toolkit redux-persist :

    /*  are used together to manage state in your React app 
      — especially in medium to large projects. */
        
       => State : The data that your app uses and changes (like user info
                theme, login, status, etc)

     (1) react-redux :
          - Think of it as a bridge between your React app and redux (which is a state
              manager.)

          - Lets your react components talk to redux
          - helps you get or update data (called 'state') from the central store.

          ex :
                import { useSelector, useDispatch } from "react-redux";


    /* Without react-redux, React and Redux can’t connect. */


      (2) @reduxjs/toolkit : 
            - This is the official, modern way to use Redux
            - Makes Redux easier to use (Redux used to be difiicult 
              before this)
            - Reduces code, handles setup for you 

            -Comes with features like : 
                  (a) createSlice() : Makes your reducers + action quickly
                  (b) configureStore() : Helps set up the store in 1 line
                  (c) createAsyncThunk() : for calling APIs (like login , register, etc)

                  Exampls : 
                     import {configureStore, createSlice } from '@reduxjs/toolkit'

       (3) redux-persist 
            - Keeps your data safe even after page refresh
            - By default, Redux store resets when you refresh the page
            - redux-persist saves data like user login, cart, theme, etc in localStorage

            Example : 
                 persistReducer(persistConfig, rootReducer); 
 
  Imagine this:
      ❤️ User logs in → App stores user data in Redux.

      ❤️ Page refreshes → Redux data is gone 😭

      ❤️ But with redux-persist, it reloads from storage automatically 😎

Example Where You’ll Use All 3:
   -If you’re building an app with:

      (1) Login/Register
      (2) Shopping Cart
      (3) Theme (dark/light) toggle
      (4) User Profile

Then these 3 help manage and preserve that data across your app easily.
